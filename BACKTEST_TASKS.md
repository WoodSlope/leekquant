# 严格回测任务池

这份任务池用于把当前“原型回测”推进到更可信的逐日历史回测。当前回测仍可用于演示流程，但不能作为真实策略收益依据。

## P0：逐日扫描主循环

- 按交易日顺序加载市场快照和股票池，而不是用当前横截面拼接历史日。
- 每个交易日执行同一套策略规则，产出当天候选信号。
- 明确当日信号生成、次日可买入、停牌或涨跌停不可成交的时间边界。
- 验收：给定固定样本行情，逐日扫描结果可重复，且不会使用未来数据。

## P0：持仓状态机

- 建立持仓生命周期：空仓、待买入、持有、待卖出、已离场。
- 同一股票在持仓期间不能重复入场；离场后是否允许重新入场要有明确规则。
- 记录入场日、入场价、仓位、持有天数和当前浮盈亏。
- 验收：多只股票、多日重叠信号下，持仓数量和每笔生命周期可追踪。

## P0：离场规则

- 将硬止损、移动止盈、跌破均线、市场系统性防御等规则改为逐日判断。
- 离场价格要使用当日可成交价格规则，避免直接用理想收益率。
- 多个离场规则同时触发时要有优先级。
- 验收：每笔交易都能解释“哪一天、哪个规则、用什么价格离场”。

## P1：收益统计

- 计算单笔收益、累计收益、最大回撤、胜率、盈亏比、持仓天数和空仓天数。
- 明确手续费、滑点、复权方式和仓位模型。
- 支持导出交易明细，便于人工复盘。
- 验收：同一输入下统计指标和交易明细一致，可用单元测试复现。

## P1：历史行情数据契约

- 统一日线字段：日期、开高低收、成交量、成交额、涨跌幅、停牌状态、涨跌停状态。
- 统一指数行情和个股行情的日期口径。
- 标记数据来源：AKShare、备用源、缓存、mock。
- 验收：缺字段、缺交易日、数据源降级时都有明确 warning，不静默污染回测。

## P2：回测 UI 升级

- 回测结果页增加交易时间轴、持仓曲线和规则命中明细。
- 区分“原型演示回测”和“严格逐日回测”。
- 在页面上显示数据源、复权、手续费、滑点等回测口径。
- 验收：用户截图时能看出回测边界，不会误认为是实盘收益。
