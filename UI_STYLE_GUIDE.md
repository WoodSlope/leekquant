# 韭菜策略 UI 规则

这份规则用于后续修改 `index.html` 时保持界面一致。当前应用是深色、克制、工具型的策略原型，不做营销页，也不把演示数据包装成真实交易建议。

## 视觉合同

- `PageHeader` 放页面级信息和页面级操作，例如监控页的 `执行扫描`。
- 页头 eyebrow 只用于必要的页面分组或状态区分；如果主标题、提示条和数据状态已经表达清楚，不额外放装饰性小标题。
- `Card` 放模块级内容。模块筛选、日期、局部操作放在卡片标题栏右侧 `actions`。
- 标题栏左边缘要和正文第一列内容左边缘对齐；右侧控件要和正文右边缘对齐。
- 表格卡片的 `headerClass` 要匹配表格单元格横向 padding：表格 `px-4` 用 `headerClass="px-4 py-3"`，表格 `px-6` 用 `headerClass="px-6 py-3"`。
- 正文为 `p-4` 的卡片，标题栏优先用 `headerClass="px-4 py-3"`。

## 控件和文案

- 选择器只显示被选择的值；没有真实多日记录时不要伪造日期下拉，“待扫描”“扫描日期”等信息用只读状态块展示。
- 按钮高度、圆角和图标节奏沿用现有按钮：主按钮用蓝色，成功态用绿色，危险态用红色。
- 不可执行但需要解释的操作可以保持可点击，并用 Toast 说明原因；真正执行中的按钮才禁用。
- 避免按钮文字换行。移动端空间不足时优先缩短文案或使用图标按钮。
- 监控页的数据源、只读、样本数据等状态统一放在页面提示条里；页头右侧只放主要操作，不额外放一个数据源提示框。
- GitHub Pages 静态快照模式不能表现成实时操作：页头按钮用灰色只读态，策略漏斗日期显示为只读观察日期，不提供日期选择器。
- 策略页“左侧选中”只是编辑态，必须持久化；只有点击应用操作才改变监控页当前运行策略。
- 监控页只展示“运行策略”，不要写成容易和策略页选中态混淆的“当前策略”；扫描完成反馈必须包含策略名和候选数量。
- 本地 API 模式下，监控页打开后先显示待扫描；只有点击“执行扫描”并收到后端结果后，才展示本地扫描信号和后端返回的扫描日期。
- 本地 API 模式下，策略漏斗标题栏只显示只读的“扫描状态 / 扫描日期”；在真正支持多日扫描记录前，不提供日期下拉。
- 信号列表的板块选择器放在该卡片标题栏右侧，只过滤表格当前显示内容；页头信号数和策略漏斗仍表示完整扫描结果。
- 扫描日期只影响策略漏斗和信号列表；“当前持仓风控”是当前模拟持仓，不随扫描日期切换为只读，只在 GitHub Pages 静态快照模式下只读。

## 数据状态

- 免费行情源必须显示来源：`akshare` 为收盘行情，`sina` 为备用收盘行情，`mock` 为样本数据。
- 当数据降级到 `mock` 时，提示应明确“样本数据 / 不适合作为真实交易依据”。
- GitHub Pages 静态模式是只读收盘复盘，页面内不能直接执行扫描。
- 本地扫描结果按运行策略保存到浏览器 IndexedDB；刷新页面或切换回该策略时恢复最近一次结果，没有记录的策略保持待扫描空态。

## 验证清单

- 桌面端和 `390px` 窄屏都要检查无横向溢出。
- 对齐问题用 `getBoundingClientRect()` 验证，严格对齐时左右差值应为 `0`。
- 关键交互至少检查一次：`执行扫描`、扫描状态显示、表格操作按钮、页面切换。
- 结束前确认控制台没有明显 `error`。
